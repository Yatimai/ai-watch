## 🔬 Recherche

### The Trinity of Consistency as a Defining Principle for General World Models
**180 upvotes** · Jingxuan Wei et al. · [Papier](https://huggingface.co/papers/2602.23152)

Propose un cadre théorique pour les World Models basé sur trois types de cohérence : modale (interface sémantique), spatiale (base géométrique) et temporelle (moteur causal). Les chercheurs introduisent CoW-Bench, un benchmark unifié pour évaluer les modèles de génération vidéo et les UMM sur des scénarios de raisonnement multi-frame. Une tentative de structurer théoriquement le domaine émergent des simulateurs de monde.

### From Blind Spots to Gains: Diagnostic-Driven Iterative Training for Large Multimodal Models  
**143 upvotes** · Hongrui Jia et al. · [Papier](https://huggingface.co/papers/2602.22859)

Méthode DPE qui diagnostique automatiquement les faiblesses des LMM puis génère des données ciblées pour les corriger, dans une boucle itérative. Tests sur Qwen3-VL-8B et Qwen2.5-VL-7B montrent des gains stables sur 11 benchmarks. Code, modèles et données disponibles publiquement. Une approche prometteuse pour l'entraînement continu et adaptatif des modèles multimodaux.

### MobilityBench: A Benchmark for Evaluating Route-Planning Agents in Real-World Mobility Scenarios
**93 upvotes** · Zhiheng Song et al. · [Papier](https://huggingface.co/papers/2602.22638)

Premier benchmark systématique pour évaluer les agents de planification d'itinéraires basés sur LLM, construit à partir de requêtes réelles d'Amap. Utilise un sandbox déterministe pour la reproductibilité et révèle que les modèles actuels peinent sur la planification avec contraintes de préférence. Benchmark et toolkit publics disponibles pour standardiser l'évaluation dans ce domaine applicatif concret.

## 🛠 Outils

### obra/superpowers — Framework agentic pour le développement logiciel  
**1546 stars/jour** · [Repo](https://github.com/obra/superpowers)

Workflow complet pour agents de code qui force une approche méthodique : extraction de spec, plan d'implémentation TDD, puis développement par sous-agents. Le système déclenche automatiquement ses "compétences" sans configuration particulière. Avec 65k stars, c'est devenu une référence pour structurer le développement assisté par IA. Jesse Vincent cherche des sponsors pour maintenir ce projet open source.

### abhigyanpatwari/GitNexus — Moteur d'intelligence de code côté client  
**1385 stars/jour** · [Repo](https://github.com/abhigyanpatwari/GitNexus)

Crée des graphes de connaissances interactifs de codebases directement dans le navigateur, avec agent RAG intégré. Fonctionne sur GitHub repos ou ZIP files, offre CLI + MCP pour connecter les agents IA aux architectures de code. Licence non-commerciale PolyForm. Concurrent de DeepWiki mais orienté analyse plutôt que compréhension, donnant aux petits modèles une vision architecturale complète.

### muratcankoylan/Agent-Skills-for-Context-Engineering — Collection de compétences pour agents  
**803 stars/jour** · [Repo](https://github.com/muratcankoylan/Agent-Skills-for-Context-Engineering)

Collection exhaustive de compétences d'agents pour l'ingénierie de contexte, architectures multi-agents et systèmes d'agents en production. Ciblé sur la gestion efficace du contexte et le debugging des systèmes d'agents. Un ressource pratique pour développeurs travaillant sur des implémentations d'agents complexes nécessitant une gestion de contexte avancée.

## 📡 Analyse

### Please, please, please stop using passkeys for encrypting user data
27 février 2026 · [Post](https://simonwillison.net/2026/Feb/27/passkeys/#atom-everything) · tags : security, usability, passkeys

Tim Cappalli met en garde contre l'utilisation des passkeys pour chiffrer les données utilisateur. Le problème : les utilisateurs perdent leurs passkeys constamment sans comprendre que leurs données sont irréversiblement chiffrées avec. Il plaide pour limiter les passkeys à leur rôle d'authentification résistante au phishing, pas au chiffrement.

### An AI agent coding skeptic tries AI agent coding, in excessive detail  
27 février 2026 · [Post](https://simonwillison.net/2026/Feb/27/ai-agent-coding-in-excessive-detail/#atom-everything) · tags : python, ai, rust, coding-agents

Max Woolf documente sa conversion aux agents de code après novembre 2025. Il décrit une progression de projets, culminant avec un port de scikit-learn en Rust. Son frustration : impossible d'expliquer les gains d'Opus 4.6/Codex 5.3 sans passer pour un "AI hype booster", alors que c'est factuellement un ordre de grandeur d'amélioration par rapport aux modèles d'il y a quelques mois.

### Free Claude Max for (large project) open source maintainers

Anthropic offre gratuitement Claude Max (plan $200/mois) aux mainteneurs open source pendant six mois, avec critères de taille de projet. Une initiative pour soutenir l'écosystème open source, mais limitée dans le temps et conditionnée à des projets "importants".
